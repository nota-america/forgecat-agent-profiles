# 07. Specialized Domains

Focused domain agents that still have a clear implementation or verification boundary.

## Tags
- Specialized Domain

## Installation
```bash
npx forgecat install @hephai-nota/awesome-codex-subagents-specialized-domains
```

## Agents
- **api-documenter** — Use when a task needs consumer-facing API documentation generated from the real implementation, schema, and examples. `gpt-5.3-codex-spark` `Specialized Domain`
- **blockchain-developer** — Use when a task needs blockchain or Web3 implementation and review across smart-contract integration, wallet flows, or transaction lifecycle handling. `gpt-5.4` `Specialized Domain`
- **embedded-systems** — Use when a task needs embedded or hardware-adjacent work involving device constraints, firmware boundaries, timing, or low-level integration. `gpt-5.4` `Specialized Domain`
- **fintech-engineer** — Use when a task needs financial systems engineering across ledgers, reconciliation, transfers, settlement, or compliance-sensitive transactional flows. `gpt-5.4` `Specialized Domain`
- **game-developer** — Use when a task needs game-specific implementation or debugging involving gameplay systems, rendering loops, asset flow, or player-state behavior. `gpt-5.4` `Specialized Domain`
- **iot-engineer** — Use when a task needs IoT system work involving devices, telemetry, edge communication, or cloud-device coordination. `gpt-5.4` `Specialized Domain`
- **m365-admin** — Use when a task needs Microsoft 365 administration help across Exchange Online, Teams, SharePoint, identity, or tenant-level automation. `gpt-5.4` `Specialized Domain`
- **mobile-app-developer** — Use when a task needs app-level mobile product work across screens, state, API integration, and release-sensitive mobile behavior. `gpt-5.4` `Specialized Domain`
- **payment-integration** — Use when a task needs payment-flow review or implementation for checkout, idempotency, webhooks, retries, or settlement state handling. `gpt-5.4` `Specialized Domain`
- **quant-analyst** — Use when a task needs quantitative analysis of models, strategies, simulations, or numeric decision logic. `gpt-5.4` `Specialized Domain`
- **risk-manager** — Use when a task needs explicit risk analysis for product, operational, financial, or architectural decisions. `gpt-5.4` `Specialized Domain`
- **seo-specialist** — Use when a task needs search-focused technical review across crawlability, metadata, rendering, information architecture, or content discoverability. `gpt-5.3-codex-spark` `Specialized Domain`
