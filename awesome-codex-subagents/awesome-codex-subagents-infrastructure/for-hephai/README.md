# 03. Infrastructure

Infrastructure-focused agents for deployment, containerization, orchestration, and IaC work.

## Tags
- Infrastructure

## Installation
```bash
npx hephai install @hephai-nota/awesome-codex-subagents-infrastructure
```

## Agents
- **azure-infra-engineer** — Use when a task needs Azure-specific infrastructure review or implementation across resources, networking, identity, or automation. `gpt-5.4` `Infrastructure`
- **cloud-architect** — Use when a task needs cloud architecture review across compute, storage, networking, reliability, or multi-service design. `gpt-5.4` `Infrastructure`
- **database-administrator** — Use when a task needs operational database administration review for availability, backups, recovery, permissions, or runtime health. `gpt-5.4` `Infrastructure`
- **deployment-engineer** — Use when a task needs deployment workflow changes, release strategy updates, or rollout and rollback safety analysis. `gpt-5.4` `Infrastructure`
- **devops-engineer** — Use when a task needs CI, deployment pipeline, release automation, or environment configuration work. `gpt-5.4` `Infrastructure`
- **devops-incident-responder** — Use when a task needs rapid operational triage across CI, deployments, infrastructure automation, and service delivery failures. `gpt-5.4` `Infrastructure`
- **docker-expert** — Use when a task needs Dockerfile review, image optimization, multi-stage build fixes, or container runtime debugging. `gpt-5.3-codex-spark` `Infrastructure`
- **incident-responder** — Use when a task needs broad production incident triage, containment planning, or evidence-driven root cause analysis. `gpt-5.4` `Infrastructure`
- **kubernetes-specialist** — Use when a task needs Kubernetes manifest review, rollout safety analysis, or cluster workload debugging. `gpt-5.4` `Infrastructure`
- **network-engineer** — Use when a task needs network-path analysis, service connectivity debugging, load-balancer review, or infrastructure network design input. `gpt-5.4` `Infrastructure`
- **platform-engineer** — Use when a task needs internal platform, golden-path, or self-service infrastructure design for developers. `gpt-5.4` `Infrastructure`
- **security-engineer** — Use when a task needs infrastructure and platform security engineering across IAM, secrets, network controls, or hardening work. `gpt-5.4` `Infrastructure`
- **sre-engineer** — Use when a task needs reliability engineering work involving SLOs, alerting, error budgets, operational safety, or service resilience. `gpt-5.4` `Infrastructure`
- **terraform-engineer** — Use when a task needs Terraform module design, plan review, state-aware change analysis, or IaC refactoring. `gpt-5.4` `Infrastructure`
- **terragrunt-expert** — Use when a task needs Terragrunt-specific help for module orchestration, environment layering, dependency wiring, or DRY infrastructure structure. `gpt-5.3-codex-spark` `Infrastructure`
- **windows-infra-admin** — Use when a task needs Windows infrastructure administration across Active Directory, DNS, DHCP, GPO, or Windows automation. `gpt-5.4` `Infrastructure`
