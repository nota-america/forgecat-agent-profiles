# PR 생성

## 개요
명확한 설명, 라벨, 리뷰어를 갖춘 잘 구조화된 풀 리퀘스트를 만들어.

## 단계
1. **변경사항 분석 및 정리**
   - 모든 코드 변경 사항 확인
   - 변경의 범위와 영향도 파악
   - 관련 이슈 또는 작업 티켓 확인

2. **PR의 이유(Why) 작성**
   - 왜 이 변경이 필요한지 명확히 설명
   - 기존 문제점 또는 개선 필요성 기술
   - 비즈니스 가치 또는 기술적 배경 포함

3. **주요 변경사항(What) 작성**
   - 핵심 변경사항만 간략하게 요약
   - 각 항목은 한 줄로 간결하게 작성
   - 브레이킹 체인지가 있다면 명시적으로 표시

4. **PR 제목 작성**
   - [티켓번호] 형식으로 시작 (예: [NPP40-483] FeignClient Redirect 허용 설정 추가)
   - 브랜치명에서 티켓 번호 확인 (예: feat/NPP40-483)
   - 변경 내용을 명확하게 표현하는 제목 작성
   - 제목만 보고도 변경 내용을 이해할 수 있도록 작성

5. **PR 메타데이터 설정**
   - 적절한 라벨 추가 (feature, bugfix, enhancement 등)
   - 관련 이슈 링크 연결
   - 적절한 리뷰어 지정
   - 필요시 마일스톤 설정

6. **시퀀스 다이어그램 생성(선택)**
   - 변경된 코드의 호출 흐름을 분석하여 Mermaid 시퀀스 다이어그램 생성
   - 단순 설정 변경, 리팩토링 등 다이어그램이 불필요한 경우 생략

7. **CLI 출력 미리보기 생성(선택)**
   - CLI 관련 파일이 변경된 경우에만 실행
   - Rich의 `Console(record=True)` + `export_svg()`로 터미널 출력 캡처
   - CLI 변경이 아닌 경우 생략

8. **참고사항 추가(선택)**
   - 관련 문서 링크
   - 참고한 이슈나 PR 링크
   - 테스트 방법이나 주의사항
   - 의존성 변경 사항

9. **검토 및 제출**
   - PR 제목과 본문을 미리보기로 보여준 뒤 `AskUserQuestion`으로 다음 액션을 물어볼 것:

   ```
   AskUserQuestion:
   - questions:
     - question: "이 내용으로 PR을 생성할까요?"
       header: "PR 생성"
       options:
         - label: "PR 생성"
           description: "위 내용으로 gh pr create를 실행합니다"
         - label: "다이어그램 추가"
           description: "시퀀스 다이어그램을 생성하여 PR 본문에 삽입합니다"
         - label: "내용 수정"
           description: "PR 제목이나 본문을 수정합니다"
         - label: "취소"
           description: "PR을 생성하지 않습니다"
       multiSelect: false
   ```

   - **"PR 생성"** 선택 시: `gh pr create`로 PR 생성
   - **"다이어그램 추가"** 선택 시: 시퀀스 다이어그램을 생성하여 본문에 삽입 후 다시 확인
   - **"내용 수정"** 선택 시: 사용자 피드백을 반영하여 재작성 후 다시 확인
   - **"취소"** 선택 시: 아무 작업도 하지 않음

## PR 템플릿

```markdown
## 🔍 PR의 이유
- ex) 외부 서비스 호출 시 기본 설정으로는 Redirect를 따르지 않아 요청 실패 발생

## ✨ 주요 변경사항
- [ ] ex) FeignClient 옵션에 Redirect 허용 설정 추가
- [ ] ex) 기타 설정 값 정리 및 주석 추가 (선택)

## 🔀 시퀀스 다이어그램(선택)

> 변경된 코드의 주요 흐름을 나타냅니다.

## 🖥 CLI Output(선택)

> CLI 명령어 변경 시, 예상되는 터미널 출력을 보여줍니다.

## 📎 참고(선택)
- ex) 관련 문서, 이슈 링크 등이 있다면 작성해주세요.
```

$ARGUMENTS 가 있으면 해당 내용을 참고하여 PR을 작성하세요.
