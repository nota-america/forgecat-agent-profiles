
## Skills

- **omni-model-explorer** — Discover and inspect Omni Analytics models, topics, views, fields, dimensions, measures, and relationships using the Omni REST API.
- **omni-query** — Run queries against Omni Analytics' semantic layer using the REST API, interpret results, and chain queries for multi-step analysis.
- **omni-model-builder** — Create and edit Omni Analytics semantic model definitions — views, topics, dimensions, measures, relationships, and query views — using YAML through the Omni REST API.
- **omni-content-explorer** — Find, browse, and organize content in Omni Analytics — dashboards, workbooks, folders, and labels — using the REST API.
- **omni-content-builder** — Create, update, and manage Omni Analytics documents and dashboards programmatically — document lifecycle, tiles, visualizations, filters, and layouts — using the REST API.
- **omni-ai-optimizer** — Optimize your Omni Analytics model for Blobby, Omni's AI assistant — configure ai_context, ai_fields, sample_queries, and create AI-specific topic extensions.
- **omni-embed** — Embed Omni Analytics dashboards in external applications — URL signing, custom themes, iframe events, entity workspaces, and permission-aware content — using the @omni-co/embed SDK and REST API.
- **omni-admin** — Administer an Omni Analytics instance — manage connections, users, groups, user attributes, permissions, schedules, and schema refreshes via the REST API.

## Agents

- **omni-analyst** — Omni Analytics data analyst — explores models, runs queries, and delivers insights.
- **omni-modeler** — Omni Analytics data modeler — builds and refines the semantic model, writes YAML, and optimizes for Blobby AI.
- **omni-admin** — Omni Analytics instance administrator — manages users, groups, permissions, schedules, connections, and access controls.

## Rules

- **omni-api-conventions** — Omni Analytics REST API conventions — authentication, base URL, error handling, and rate limits.
- **omni-yaml-conventions** — Omni Analytics model YAML conventions — file types, syntax patterns, field references, and common pitfalls.
- **omni-terminology** — Omni Analytics terminology and concept mapping — translates business intelligence terms to Omni-specific vocabulary.
